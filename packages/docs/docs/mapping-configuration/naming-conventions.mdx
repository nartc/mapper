---
title: namingConventions
description: Override source and destination naming conventions for one mapping.
---

```ts
createMap(
  mapper,
  User,
  UserDto,
  namingConventions(new CamelCaseNamingConvention()),
);
```

Use separate conventions when the source and destination differ:

```ts
createMap(
  mapper,
  ProductRow,
  ProductDto,
  namingConventions({
    source: new SnakeCaseNamingConvention(),
    destination: new CamelCaseNamingConvention(),
  }),
);
```

Mapping-level configuration overrides the convention supplied to `createMapper()` or `addProfile()`. See [Naming conventions](/fundamentals/naming-convention) for mapper- and profile-level examples.
